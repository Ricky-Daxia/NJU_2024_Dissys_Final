# 课程 PPT 整体理解与补充

这份文件用于对照 `PPT及中文教材` 里的课程 PPT，检查现有复习材料是否覆盖课程真实讲法。结论是：现有材料已经覆盖往年题高频算法和最新复习 PDF 的主线，但课程 PPT 里还有一些更像“课堂概念问答”的细节，应该作为补充分背熟。

## 1. 课程主线

| PPT | 主题 | 复习定位 |
| --- | --- | --- |
| 01-introduction | 分布式系统定义、目标、透明性、开放性、可扩展性、系统类型 | 概念问答基础 |
| 02-architecture | 架构风格、集中/去中心/混合组织、C/S、P2P、中间件、自管理系统 | 概念问答 + 架构分析 |
| 03-processes | 进程、线程、服务端组织、虚拟化、代码迁移 | 概念问答 |
| 04-1-communication | 分层协议、RPC、参数传递、动态绑定、RPC 故障语义 | 高频大题 |
| 04-2-communication | 消息通信、消息队列、消息代理、流数据、QoS、应用层组播 | 补分问答 |
| 06-Synchronization | 物理时钟、Lamport、向量时钟、因果组播、互斥、选举 | 算法计算 + 问答 |
| 07-Consistency & replication | 一致性模型、客户为中心一致性、复制协议、Quorum | 高频判断题/大题 |
| 08-Fault Tolerance | 可信系统、故障模型、冗余、进程组、可靠/原子组播、2PC/3PC、恢复 | 高频大题 |

课程整体不是只讲“算法”，而是按这个逻辑展开：

```text
为什么需要分布式系统
-> 如何组织成一个系统
-> 进程之间如何通信
-> 没有全局时钟时如何同步
-> 多副本如何保持一致
-> 有故障时如何继续服务和恢复
```

## 3. 需要补充的 PPT 细节

### 3.1 基础与架构

- 可扩展性分三类：规模可扩展、地理可扩展、管理可扩展。
- 透明性不是越强越好，过度隐藏可能增加复杂度、降低性能或让错误更难处理。
- 架构风格要会说：分层、对象/服务式、发布订阅、共享数据空间。
- P2P 除结构化/非结构化外，PPT 还强调 super-peer 和混合式组织。
- 中间件里要知道 interceptor：在调用路径中插入通用逻辑，例如安全、压缩、日志、协议转换。
- 自管理系统可按反馈控制模型回答：监控、分析、计划、执行。

### 3.2 进程与通信

- 线程实现要能区分 user-level thread、kernel-level thread 和 mixed approach。
- 服务端状态要能区分 stateless、stateful、soft-state。
- 虚拟化要会区分 process virtual machine 和 virtual machine monitor。
- 消息代理用于在异构系统之间转换消息格式和路由消息。
- 流数据不只看是否到达，还看带宽、延迟、抖动、同步和 QoS。
- 应用层组播可用 link stress 和 stretch 描述代价。

### 3.3 同步与复制

- Berkeley 算法：时间守护进程收集各节点时间，计算平均偏差，再通知各节点调整。
- NTP：分层时间服务器体系，目标是让互联网规模系统近似同步。
- 因果有序组播：若消息 `m1` 因果先于 `m2`，任何进程都不能先交付 `m2` 再交付 `m1`。
- 复制协议实例要会说：主备复制、主动复制、链式复制、ROWA、Quorum。

### 3.4 容错与恢复

- 故障模型要会列：crash、omission、timing、response、arbitrary/Byzantine。
- 冗余类型要会列：信息冗余、时间冗余、物理冗余。
- 进程组要会区分 flat group 和 hierarchical group。
- 可靠组播保证消息最终到达所有非故障成员；原子组播还要求所有成员以相同顺序交付。
- 2PC 参与者进入 ready/prepared 后，若没有最终决定，会发生阻塞。
- 副本数公式必须区分场景：crash/fail-silent 通常 `K+1`，唯一正确结果多数投票是 `2K+1`，拜占庭一致是 `3K+1`。
